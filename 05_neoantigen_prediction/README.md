# 05 — Neoantigen Prediction

## The Core Problem

This is the most important step in the pipeline and the one with the biggest unsolved problem.

**What we're doing:** Given somatic mutations in a tumor and the patient's HLA type, predict which mutant peptides will be presented on the cell surface and recognised by T-cells.

**Why it matters:** If we can identify the right neoantigens, we can design a vaccine that trains the patient's immune system to attack their specific cancer. If we pick the wrong ones, the vaccine is wasted.

**The problem:** Current tools (pVACseq, NetMHCpan) predict MHC *binding* well, but binding is only the first step. A peptide must:

1. Be generated by proteasomal cleavage (processing)
2. Be transported to the ER by TAP (transport)
3. Bind the patient's MHC-I molecule (binding) ← current tools are good here
4. Be stable enough on the surface (stability)
5. Be recognised by a T-cell receptor (TCR recognition) ← nobody predicts this well
6. Trigger T-cell activation, not tolerance (immunogenicity) ← the real question

Steps 1-3 are reasonably well-modeled. Steps 4-6 are where the false positives come from.

## Tools

- **pVACseq** — orchestrates the neoantigen prediction pipeline
- **NetMHCpan-4.1** — neural network for MHC-I binding prediction
- **MHCflurry** — alternative binding predictor (ensemble approach)

## Key Parameters

- **Epitope length:** 8-11 amino acids for MHC-I (9 is most common)
- **Binding threshold:** IC50 < 500nM (standard), < 50nM (strong binder)
- **Fold change:** mutant binding / wildtype binding (higher = more tumour-specific)

## The Research Frontier

The gap between "binds MHC" and "triggers immune response" is where novel scoring approaches are needed. This is the substrate for:

- Structural approaches (how different does the mutant look from self?)
- Information-theoretic approaches (how much "surprise" does the mutation introduce?)
- Cross-scale models (how does sequence-level change propagate to immune outcomes?)

See `rank_candidates.py` for the current heuristic scoring (v0) and the placeholder for entropy-aware scoring (v1).
